# 架构说明

## 模块职责

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐
│   gui.py    │────▶│ generator.py │────▶│  output.xlsx │
│  (界面交互)  │     │  (生成Excel)  │     │              │
└──────┬──────┘     └──────┬───────┘     └──────────────┘
       │                   │
       │            ┌──────▼───────┐
       │            │template_     │
       │            │reader.py     │
       │            │(解析模板格式) │
       │            └──────────────┘
       │
       ▼
┌──────────────┐
│  parser.py   │
│ (文本→结构化) │
└──────────────┘
```

### template_reader.py
- 用 openpyxl 读取模板 .xlsx
- 提取：列数、表头文本（第4行）、列宽、合并单元格、字体、对齐、边框、行高
- 返回 `TemplateFormat` 对象

### generator.py
- 接收 TemplateFormat + 用户数据列表
- 按模板格式新建 .xlsx（列结构、样式、公式）
- 插入图片到图片列
- 公式：总数量 = 件数 × 每件数量，总金额 = 总数量 × 单价

### parser.py
- 将用户粘贴的文本解析为结构化数据
- 支持三种格式：表格（Excel复制）、多行key-value、单行key-value
- 自动提取：品名、单价、件数、每件数量、重量等字段

### gui.py
- Tkinter 主窗口
- 顶部工具栏：选择模板、选择照片
- 左侧：照片缩略图列表
- 右侧：文本框（粘贴产品信息）
- 底部：生成按钮

### main.py
- 启动 GUI

## 数据流

1. 用户选择模板 → template_reader 解析 → 得到 TemplateFormat
2. 用户选择照片文件夹 → gui 列出所有照片
3. 用户粘贴文本 → parser 解析为结构化数据列表
4. 点击生成 → generator 接收 TemplateFormat + 数据列表 → 输出 .xlsx

## 关键约定

- 模板第4行为表头行（与原始模板一致）
- 图片统一插入第一列
- 未识别的列留空
- 公式列自动生成
